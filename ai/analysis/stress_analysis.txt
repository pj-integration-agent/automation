Relatório Técnico: Teste de Estresse na API PokéAPI

Resumo do Cenário de Teste:
O teste de estresse realizado teve como objetivo avaliar o desempenho da API PokéAPI ao realizar requisições do tipo GET para obter informações sobre o Pokémon "Ditto". O teste foi executado com 20 threads concorrentes, simulando uma carga de usuários acessando a aplicação.

Volume Total de Requisições:
O teste gerou um total de 20 requisições GET para a API PokéAPI.

Throughput Estimado:
O throughput estimado, calculado com base no tempo total de execução do teste e no número de requisições bem-sucedidas, é de aproximadamente 0,65 requisições por segundo.

Latência:
- Latência p50 (mediana): 21 ms
- Latência p90: 27 ms
- Latência p95: 28 ms
- Latência p99: 928 ms

Taxa de Erros:
Todas as 20 requisições tiveram sucesso, com código de resposta 200 OK. Não houve nenhuma falha registrada.

Principais Gargalos Identificados:
Não foram identificados gargalos significativos no desempenho da API durante o teste. As latências observadas estão dentro de valores aceitáveis para a maioria das aplicações.

Impacto Potencial em Ambiente de Produção:
Considerando o baixo volume de requisições e o alto desempenho observado, o impacto da carga de teste no ambiente de produção seria mínimo. A API PokéAPI demonstrou capacidade de atender a esse nível de tráfego sem comprometer a experiência do usuário.

Recomendações Técnicas:
1. Continuar monitorando o desempenho da API PokéAPI em ambiente de produção, especialmente durante períodos de maior demanda.
2. Implementar testes de carga periódicos para identificar possíveis regressões de desempenho à medida que a aplicação evolui.
3. Considerar realizar testes com cargas mais elevadas, simulando cenários de pico de uso, a fim de identificar os limites de capacidade da API.
4. Avaliar a possibilidade de implementar mecanismos de cache e otimizações adicionais na API, visando reduzir ainda mais a latência das respostas.
5. Manter a documentação atualizada e acessível para que a equipe de desenvolvimento possa compreender melhor a API e suas características de desempenho.
