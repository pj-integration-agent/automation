**User Story 01 - Criar Novo Banco**

Como um QA especialista, eu quero poder criar um novo registro de banco para que seja possível adicionar informações de novos bancos no sistema.

Descrição:
- Ao clicar na opção "Novo", o usuário deve ser direcionado para a tela de cadastro de banco.
- Os campos obrigatórios a serem preenchidos são: "Código", "Descrição do Banco", "Apelido", "Número de inscrição no SBP", "Banco controlador" (com opções em uma lista) e CNPJ (que deve ser válido e existente).
- O campo "Código" não pode ser igual a outro já cadastrado em outros registros.
- Após preencher todos os campos obrigatórios, o usuário deve clicar no botão "Salvar" para validar o cadastro do novo banco.

Critérios de Aceitação:
- Todos os campos obrigatórios devem ser preenchidos corretamente.
- O campo "Código" não pode ser igual a outro já cadastrado.
- O CNPJ informado deve ser válido e existente.
- Ao clicar em "Salvar", o novo banco deve ser cadastrado com sucesso.

Rastreabilidade: US01

**User Story 02 - Editar Registro de Banco**

Como um QA especialista, eu quero poder editar as informações de um banco já cadastrado, exceto o campo "Código", para que seja possível manter os registros atualizados.

Descrição:
- Ao selecionar um banco da lista de registros existentes, o usuário deve poder abrir o formulário de edição.
- Os campos editáveis são: "Descrição do Banco", "Apelido", "Número de inscrição no SBP", "Banco controlador" e CNPJ.
- O campo "Código" não pode ser editado.
- Após realizar as alterações desejadas, o usuário deve clicar no botão "Salvar" para validar as mudanças.

Critérios de Aceitação:
- Todos os campos editáveis devem poder ser alterados, exceto o "Código".
- As alterações realizadas devem ser salvas corretamente ao clicar em "Salvar".
- Não deve ser possível alterar o valor do campo "Código".

Rastreabilidade: US02
