Com base na imagem da página web do ParaBank, podemos gerar as seguintes User Stories:

US01: Como cliente, eu quero poder fazer login na minha conta para acessar meus serviços bancários online.
- Descrição: A página web deve apresentar um formulário de login para que os clientes possam inserir seu nome de usuário e senha e acessar sua conta.
- Critérios de Aceitação:
  - O campo de nome de usuário deve aceitar apenas entradas válidas.
  - O campo de senha deve aceitar apenas entradas válidas e estar mascarado.
  - Ao clicar no botão "LOG IN", o sistema deve autenticar as credenciais e conceder acesso à conta do cliente.
  - Caso as credenciais sejam inválidas, o sistema deve exibir uma mensagem de erro amigável.

US02: Como cliente, eu quero ter acesso a serviços bancários básicos através de minha conta online para realizar transações financeiras.
- Descrição: A página web deve apresentar uma seção de "Serviços Online" com as principais funcionalidades bancárias, como saque, transferência de fundos, consulta de saldo e histórico de conta.
- Critérios de Aceitação:
  - A seção de "Serviços Online" deve listar as opções de "Bill Pay", "Account History", "Transfer Funds" e outras funcionalidades relevantes.
  - Ao clicar em cada opção, o sistema deve direcionar o cliente para a tela correspondente e permitir a realização da transação.
  - O sistema deve validar as informações inseridas pelo cliente e exibir mensagens de confirmação ou erro, conforme apropriado.

US03: Como cliente, eu quero ter acesso a serviços bancários básicos através de caixas eletrônicos (ATM) para realizar transações financeiras presencialmente.
- Descrição: A página web deve apresentar uma seção de "Serviços ATM" com as principais funcionalidades bancárias disponíveis nos caixas eletrônicos, como saque, transferência de fundos, consulta de saldo e depósitos.
- Critérios de Aceitação:
  - A seção de "Serviços ATM" deve listar as opções de "Withdraw Funds", "Transfer Funds", "Check Balances" e "Make Deposits".
  - Ao clicar em cada opção, o sistema deve fornecer instruções claras sobre como realizar a transação nos caixas eletrônicos.
  - O sistema deve validar as informações inseridas pelo cliente e exibir mensagens de confirmação ou erro, conforme apropriado.

US04: Como cliente, eu quero ter acesso a notícias e atualizações sobre a instituição bancária para ficar informado sobre mudanças e novidades.
- Descrição: A página web deve apresentar uma seção de "Últimas Notícias" com informações relevantes sobre a ParaBank, como reaberturas de agências, novos serviços online e outras atualizações importantes.
- Critérios de Aceitação:
  - A seção de "Últimas Notícias" deve exibir informações atualizadas e relevantes para os clientes.
  - Ao clicar na opção "READ MORE", o sistema deve direcionar o cliente para uma página com mais detalhes sobre a notícia.
  - O sistema deve manter a seção de notícias atualizada com as informações mais recentes.
