Com base na imagem da plataforma web do ParaBank, podemos gerar as seguintes User Stories completas:

US01: Como cliente do ParaBank, eu quero poder fazer saques de fundos na rede de caixas eletrônicos para ter acesso ao meu dinheiro quando necessário.
Descrição: O usuário deve poder inserir seu cartão, digitar a senha e selecionar a opção de saque para retirar uma quantia de sua conta bancária.
Critérios de Aceitação:
- O sistema deve autenticar o usuário com base no cartão e senha fornecidos.
- O sistema deve exibir o saldo disponível na conta do usuário.
- O sistema deve permitir que o usuário selecione o valor a ser sacado.
- O sistema deve processar o saque e atualizar o saldo da conta.
- O sistema deve emitir um comprovante da transação.

US02: Como cliente do ParaBank, eu quero poder transferir fundos entre minhas contas para gerenciar melhor meus recursos financeiros.
Descrição: O usuário deve poder selecionar a conta de origem, digitar o valor a ser transferido e escolher a conta de destino para realizar a transferência.
Critérios de Aceitação:
- O sistema deve autenticar o usuário com base no cartão e senha fornecidos.
- O sistema deve exibir o saldo disponível nas contas do usuário.
- O sistema deve permitir que o usuário selecione a conta de origem e de destino.
- O sistema deve permitir que o usuário informe o valor a ser transferido.
- O sistema deve processar a transferência e atualizar os saldos das contas envolvidas.
- O sistema deve emitir um comprovante da transação.

US03: Como cliente do ParaBank, eu quero poder consultar o saldo de minhas contas para acompanhar minha situação financeira.
Descrição: O usuário deve poder selecionar a conta desejada e visualizar o saldo atualizado.
Critérios de Aceitação:
- O sistema deve autenticar o usuário com base no cartão e senha fornecidos.
- O sistema deve exibir a lista de contas do usuário.
- O sistema deve permitir que o usuário selecione a conta desejada.
- O sistema deve exibir o saldo atualizado da conta selecionada.
